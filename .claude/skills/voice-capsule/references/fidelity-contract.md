# Fidelity Contract (v0.1)

윤문본은 다음을 token level 로 보존한다. 위반 edit 만 롤백.

## 보존 대상

1. 수치. 아라비아·한자·문자형 숫자 모두. 단위 (°C, %, kg, 명, 건, MW) 포함.
2. 날짜·시간. "2026년 5월", "오후 3시", "분기" 등 명시 시점.
3. 고유명사. 인명·기관명·제품명·지명·법령명. 한자 병기 포함.
4. 영어 약어·식별자. LLM, GPU, API, MCP, IPv6, KAERI, K-방산 등.
5. 큰따옴표 직접인용. `"..."` 안쪽 한 글자도 바뀌지 않는다.
6. 법률 조문 인용. `○○법 제○조 ○항` 형식.
7. 수식·코드. 인라인 코드, 수식, 코드블록.
8. 사용자 지정 `donot_keywords` 의 모든 토큰.

## 토큰화

- 화이트스페이스 + 문장부호 기준 분할.
- 한자·숫자 클러스터는 하나의 토큰.
- 큰따옴표 직접인용은 인용 전체가 단일 보존 단위.

## 검증 알고리즘

```
1. P ← 원문에서 보존 대상 토큰 집합 추출
2. Q ← 윤문본 토큰 집합 추출
3. P ⊆ Q 여야 한다
4. P \ Q (윤문본에서 사라진 보존 토큰) 비어 있지 않으면 → 위반
5. 위반 edit 1건 → 그 edit 만 롤백 후 1회 재시도
6. 재시도 후에도 위반 → 그 구간은 원문 그대로 두고 VC-SUMMARY 의 flags 에 기록
```

## 변경률 가드

```
변경률 ≈ 1 - LCS(원문, 윤문본).length / max(|원문|, |윤문본|)
```

- 변경률 ≤ 30% → 정상.
- 30% < 변경률 ≤ 50% → `over_polish_warning` 플래그.
- 50% < 변경률 → 강제 중단, 직전 안전 버전 (≤30%) 출력, `over_polish_aborted` 플래그.

문자 단위·어절 단위 중 어느 쪽으로 계산하든 한 캡슐 안에서 일관성만 유지하면 된다.
