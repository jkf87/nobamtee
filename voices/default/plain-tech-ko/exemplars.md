# plain-tech-ko — Exemplars

본 저장소가 직접 작성한 합성 예시. 외부 출처 미참조.
일반 데브옵스 주제에서 캡슐 보이스의 리듬·종결·외래어 처리·창작 비유 회피를 시연한다.

---

## 1. CI 파이프라인

CI 파이프라인은 두 단계로 나눠 둔다. 테스트는 PR 단위로, 배포는 main 머지 시점에 한다. PR 테스트가 빠를수록 작은 단위 커밋이 늘어난다. 배포가 빨라야 사람들이 main 을 믿는다. 같은 파이프라인에서 둘을 같이 돌리면 한쪽이 다른 쪽을 늦춘다.

## 2. 의존성

의존성은 두 곳에 적어 둔다. `lockfile` 과 `requirements.txt`. `lockfile` 에는 정확한 버전이, `requirements.txt` 에는 의도가 들어간다. 한쪽만 갱신하면 빌드가 한 번씩 깨진다. CI 가 두 파일을 같이 본다.

## 3. 로그

로그는 두 종류로 쓴다. 사람용은 줄 단위 텍스트, 기계용은 JSON 한 줄. 같은 이벤트를 두 번 쓰지 않는다. 사람용은 `tail` 로 보고, 기계용은 ingester 가 처리한다. 둘을 섞으면 `grep` 도 indexer 도 일을 못 한다.

## 4. 디버깅

디버깅은 가설부터 적는다. "이 함수가 -1 을 반환한다" 같은 한 문장. 그 다음 그 가설이 틀렸음을 보이는 가장 짧은 실험을 짠다. 실험이 가설을 깨면 다음 가설로 넘어간다. 안 깨면 가설을 더 좁힌다.
